====== 1. bases ======

===== Config =====
**Trois niveau**
* system : dans ''/etc/gitconfig''
* global (user) : dans ''~/.gitconfig''
* repository : dans ''.git/config'' du repository

**Exemple d'initialisation au niveau global (user)**
''git config --global user.name "John Doe"''
''git config --global user.email johndoe@example.com''

→ au niveau système, il faut utiliser ''--system''
→ au niveau repository, ne pas mettre de flag spécifique

Pour l'éditeur de texte : ''git config --global core.editor emacs''

**Lire une valeur**
''git config user.name''

===== Commandes de base =====
**git init**
Initialiser un repository. créé le dossier ''.git''

**git clone**
* ''git clone https://github.com/gandrille/aws-glacier-cli.git''
* ''git clone git@github.com:gandrille/aws-glacier-cli.git''
* ''git clone utilisateur@serveur:/chemin.git''

**git add**
Ajouter un fichier ou un dossier (tous ses fichiers récursivement) à l'index
* ''git add foo.txt''
* ''git add *.c''
* ''git add src/''
* ''git add .''

**git rm**
* ''git rm foo.txt'' : supprime le fichier suivi (si ce n'est déjà fait) et ajoute la suppression à l'index
* ''git rm --cached foo.txt'' : retire le fichier foo.txt de l'index. Par exemple quand on a fait un ''add'' par erreur.

**git mv**
Commande de convenance, qui fait la même chose que :
'''
mv source destination
git rm source
git add destination
'''

**gitstatus**
Pour connaître l'état de l'espace de travail 
''git status -s'' ou ''git status --short'' permet d'avoir les infos sur une seule ligne

**git commit**
* ''-m'' : pour ajouter le message sur la ligne…
* ''-a'' : 

**git checkout **(Update le working directory)
Déplace HEAD
* ''git checkout branchname''
* ''git checkout commitid''
Sans déplacer HEAD
* ''git checkout foo.txt'' Supprime les modifications de la copie de travail. L'état obtenu est celui de l'index ou du dernier commit.

===== Dépots distants =====
* ''git remote [-v]'' : liste les serveurs distants enregistrés. ''origin'' est automatiquement enregistré lors d'un ''git clone''
* ''git remote show origin''
* ''git remote add [nomcourt] [url]''
* ''git remote rename [oldname] [newname]''
* ''git remote rm [name]''

===== Tags =====
**Lister par ordre alphabétique (et NON chronologique)**
* ''git tag''
* ''git tag -l "v2019*"''
* ''git show tagname''

**Tags légers (lightweight)**
* Autre nom pour un commit
* Création : ''git tag hash''

**Tag annoté**
* créé un commit pour le tag. On a donc un nom de tag, et un message de commit et une date de commit… le tout référençant un autre commit
* Création : ''git tag -a nom -m 'mon message de commit' hash''

**Pousser un tag**
Par défaut, les tags restent locaux !
* Pour publier un tag, il faut faire : ''git push origin montag''
* Pour publier tous les tags : ''git push origin --tags''
